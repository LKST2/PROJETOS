PROCESSO:

1. Alteração do código;

2. Adição de TODOS os arquivos alterados com gti add . ;

3. Registro de informação sobre a mudança criada: git commit -m "mensagem"

4. Envio para nuvem git push

CONCEITOS DE BRANCH:

Basicamente é um conceito de operação e desenvolvimento de versãos sem ter de estagnar a a cadeia principal de produção. Uma espécie de manutenção do backup.

Git Merge - unir informaçoes de uma branch à outra. **BOA PRÁTICA** para não correr o risco de de jogar uma atualização para versão antiga traga as informações da branch a ser atualizada para máquina local e dpois incorpore a nova versão. E aí suba novamente o código já atuallizado:

*git pull - traz o código numa branch emporária para local
**git merge nomedabranchprincipal - vai unir a branch modifica à principal
***git push - para atualizar na nuvem
